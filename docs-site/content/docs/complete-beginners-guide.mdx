## What is Robot Programming?

Let's start with the basics.

### Real Robots Have Multiple Parts

A robot isn't one big program - it's many small programs working together:

- **Sensors** - Read data (camera, distance sensors, GPS)
- **Processors** - Make decisions (AI, navigation, control)
- **Actuators** - Take action (motors, servos, grippers)

Each part needs to **talk to the others** super fast. That's what HORUS does - it makes all your robot parts communicate instantly.

### The Challenge: Making Parts Talk

Imagine building a self-driving car. You need:

1. Camera reading the road (30 times per second)
2. AI deciding where to steer
3. Motors turning the wheels
4. Safety system that can STOP INSTANTLY

All of these need to share data in **milliseconds** or someone gets hurt. HORUS makes this trivial.

### HORUS Makes This Easy

Instead of worrying about how to send data between components, you just:

```python
# Component 1: Read sensor
node.send("distance", 1.5)

# Component 2: Receive data
distance = node.get("distance")
```

That's it. HORUS handles all the complexity for you.

## Part 2: Your First Robot Program (15 Minutes)

Let's build a simple robot that moves in a square pattern. We'll start in a simulator so you don't need actual hardware.

### Understanding Robot "Nodes"

In HORUS, we call each robot component a **"node"**. Think of nodes like Lego blocks:

- Each node does ONE job
- Nodes connect together by sending messages
- You build complex robots by combining simple nodes

**Example nodes:**
- `CameraNode` - Reads images from a camera
- `MotorNode` - Controls motors
- `AINode` - Makes decisions

### Create Your First Project

```bash
horus new my_first_robot
cd my_first_robot
```

HORUS creates a folder with everything set up for you.

### Understanding the Code

HORUS created a file called `main.py`. Let's look at it:

```python
import horus

# Counter to track messages
count = 0

# This function runs repeatedly - the "heartbeat" of your robot
def tick(node):
    global count
    count += 1
    print(f"Hello from robot! Count: {count}")

    # Stop after 5 messages
    if count >= 5:
        node.request_stop()

# Create and run the robot
node = horus.Node(name="HelloRobot", tick=tick, rate=1)  # 1 Hz = once per second
horus.run(node, duration=5)
```

**What's happening here?**

1. `horus.Node()` - Creates a robot component
2. `tick=tick` - Function that runs repeatedly (like a heartbeat)
3. `rate=1` - Run once per second (1 Hz)
4. `horus.run()` - Start the robot!
5. `duration=5` - Run for 5 seconds then stop

### Run Your First Robot

```bash
horus run
```

You'll see:
```
Hello from robot! Count: 1
Hello from robot! Count: 2
Hello from robot! Count: 3
Hello from robot! Count: 4
Hello from robot! Count: 5
```

**Congratulations! You just programmed your first robot component!**

## Part 3: Making Nodes Talk (Communication)

Real robots need multiple components working together. Let's build a simple example:

- **SensorNode** - Simulates reading distance from an obstacle
- **MotorNode** - Decides whether to move forward or stop

### Create a New File: `robot_with_sensor.py`

```python
import horus
import random

# Node 1: Reads distance sensor
def sensor_tick(node):
    # Simulate reading a sensor (random distance 0-10 meters)
    distance = random.uniform(0.5, 10.0)

    # Send the distance to anyone listening
    node.send("distance", distance)
    print(f"Sensor: Obstacle at {distance:.2f}m")

# Node 2: Controls motors based on sensor data
def motor_tick(node):
    # Receive distance data
    distance = node.get("distance")

    if distance is not None:
        if distance < 1.0:
            print(f"  Motor: STOP! Too close ({distance:.2f}m)")
        else:
            print(f"  Motor: Moving forward (clear at {distance:.2f}m)")

# Create both nodes
sensor = horus.Node(name="DistanceSensor", pubs=["distance"], tick=sensor_tick, rate=2)
motor = horus.Node(name="MotorController", subs=["distance"], tick=motor_tick, rate=2)

# Run both nodes together for 5 seconds
horus.run(sensor, motor, duration=5)
```

### Run It

```bash
python3 robot_with_sensor.py
```

**Output:**
```
Sensor: Obstacle at 7.23m
  Motor: Moving forward (clear at 7.23m)
Sensor: Obstacle at 0.84m
  Motor: STOP! Too close (0.84m)
Sensor: Obstacle at 5.67m
  Motor: Moving forward (clear at 5.67m)
```

### What Just Happened?

1. **SensorNode** reads distance and publishes to `"distance"` topic
2. **MotorControllerNode** subscribes to `"distance"` topic and receives the data
3. Motor makes decisions based on sensor data
4. **All communication happens in 248 nanoseconds** (0.000248 milliseconds)

**This is the foundation of ALL robot programming.** Sensors  Processing  Actuators.

## Part 5: Build a Real Robot (Next Steps)

Now that you understand the basics, here's how to build actual robots:

### For Hardware Robots

HORUS works with:
- **Raspberry Pi** robots
- **Arduino** via serial communication
- **Jetson Nano** for AI robots
- Any **Linux-based** robot controller

**Example hardware setup:**
```python
import horus
import RPi.GPIO as GPIO  # Raspberry Pi GPIO library

# Setup GPIO pins for motor driver
GPIO.setmode(GPIO.BCM)
GPIO.setup(17, GPIO.OUT)  # Left motor
GPIO.setup(18, GPIO.OUT)  # Right motor

def motor_tick(node):
    # Receive velocity commands
    velocity = node.get("cmd_vel")
    if velocity:
        linear, angular = velocity
        # Convert to motor speeds and drive!
        left_speed = linear - angular
        right_speed = linear + angular
        # Write to GPIO (simplified example)
        GPIO.output(17, left_speed > 0)
        GPIO.output(18, right_speed > 0)

# Create and run the motor controller node
motor = horus.Node(
    name="MotorController",
    subs=["cmd_vel"],
    tick=motor_tick,
    rate=50  # 50 Hz control loop
)

horus.run(motor)
```

## Why HORUS is 500x Faster

You keep seeing "500x faster than ROS2" - what does that mean?

### The Speed Comparison

| Task | HORUS | ROS2 | Improvement |
|------|-------|------|-------------|
| Send message between nodes | 248ns | 156,000ns | **500x faster** |
| Camera  AI  Motors | 2.1ms | 847ms | **403x faster** |
| Emergency STOP reaction | 0.8ms | 125ms | **156x faster** |

### Why Speed Matters

**Example: Self-Driving Car**

Imagine you're going 60 mph (26.8 m/s):

- **With ROS2**: 156Âµs latency = **4.2 meters** traveled before reacting
- **With HORUS**: 248ns latency = **0.008 meters** (8mm) before reacting

**HORUS stops 525x sooner.** In robotics, microseconds save lives.

### How HORUS Achieves This

**Traditional frameworks (ROS2)**:
1. Serialize data to bytes
2. Send over network sockets
3. Deserialize on the other end
4. Copy data multiple times

**HORUS**:
1. Write directly to shared memory
2. Reader reads directly (zero-copy)
3. Done!

No serialization. No network. No copying. Just pure speed.

## Common Beginner Questions

### Q: Do I need to learn Rust?

**No!** Use Python for everything. Only learn Rust if you need maximum performance later.

### Q: Do I need a robot to get started?

**No!** Use the simulator (`horus sim --2d`) to learn. Build programs that work in simulation, then deploy to real hardware when ready.

### Q: Can I use HORUS for drones?

**Yes!** HORUS works great for drones, especially for:
- Flight controllers
- Autonomous navigation
- Computer vision (obstacle detection)
- Swarm coordination (multiple drones)

### Q: How is this different from Arduino?

**Arduino**: Great for simple sensors and motors, but limited computing power. Code runs on tiny microcontroller.

**HORUS**: Runs on Linux computers (Raspberry Pi, Jetson, laptop). Much more powerful - can run AI, cameras, complex navigation.

**Best combo**: Use Arduino for low-level hardware, HORUS for high-level intelligence.

### Q: Can HORUS control multiple robots?

**Yes!** You can run HORUS on multiple robots and have them communicate:

```python
# Robot 1: Coordinator node
def coordinator_tick(node):
    positions = [(x1, y1), (x2, y2), (x3, y3)]
    node.send("swarm/positions", positions)

# Robot 2, 3, 4: Worker nodes subscribe to the same topic
def worker_tick(node):
    swarm_positions = node.get("swarm/positions")
    if swarm_positions:
        # React to swarm positions
        pass
```

## Your First Real Project: Build a Line Following Robot

Ready to build something real? Here's a complete beginner project.

### What You'll Build

A robot that follows a black line on the floor using IR sensors.

**Hardware needed:**
- Robot chassis with 2 motors (~$25)
- Raspberry Pi or Arduino (~$35)
- IR line sensors x2 (~$5)
- Motor driver (~$8)

**Total cost: ~$75**

### The Code (Complete)

```python
import horus

# TODO: Import your GPIO library (e.g., RPi.GPIO)

# Node 1: Reads left and right IR sensors
def sensor_tick(node):
    # Read sensors (True = black line detected)
    # TODO: Replace with actual GPIO reads
    left_sees_line = read_left_sensor()   # Placeholder function
    right_sees_line = read_right_sensor() # Placeholder function

    node.send("sensor/left", left_sees_line)
    node.send("sensor/right", right_sees_line)

# Node 2: Controls motors to follow the line
def follower_tick(node):
    left = node.get("sensor/left")
    right = node.get("sensor/right")

    if left is None or right is None:
        return

    # Line following logic
    if left and right:
        # Both sensors on line - go straight
        node.send("cmd_vel", (1.0, 0.0))
    elif left:
        # Left sensor on line - turn left
        node.send("cmd_vel", (0.5, -0.3))
    elif right:
        # Right sensor on line - turn right
        node.send("cmd_vel", (0.5, 0.3))
    else:
        # Lost the line - stop and search
        node.send("cmd_vel", (0.0, 0.5))

# Node 3: Drives the actual motors
def motor_tick(node):
    cmd = node.get("cmd_vel")
    if cmd:
        linear, angular = cmd
        # Convert to left/right motor speeds
        left_speed = linear - angular
        right_speed = linear + angular
        # TODO: Write to motor driver
        drive_motors(left_speed, right_speed)  # Placeholder function

# Create nodes
sensor = horus.Node(name="LineSensor", pubs=["sensor/left", "sensor/right"], tick=sensor_tick, rate=30)
follower = horus.Node(name="LineFollower", subs=["sensor/left", "sensor/right"], pubs=["cmd_vel"], tick=follower_tick, rate=30)
motor = horus.Node(name="MotorDriver", subs=["cmd_vel"], tick=motor_tick, rate=30)

# Run all nodes
horus.run(sensor, follower, motor)
```

**That's it!** Three nodes, ~50 lines of code, and you have a line following robot.
